# 활용: 완성된 지식베이스를 "보고서 작성 기준"으로 쓰는 법

> 가공이 끝난 자료(`samples/*_processed.json`)가 실제 **보고서·입찰제안서 작성의 기준선(baseline)**이
> 되는 방식. 핵심: 모든 주장에 **`법령명 + 조항 + 시행일 + 출처URL`**이 박혀 나온다.

---

## 큰 그림

지식베이스 = "이 사안의 근거 규정이 무엇이고, 지금 유효한가"를 즉시 답하는 **규정 기준선**.
보고서를 쓰는 흐름에 3가지로 끼어든다.

```
            ┌── ① 작성 중: 근거 즉시 조회·인용 (search/report)
보고서 작성 ─┼── ② 작성 후: 규정 준수 점검 (대조)
            └── ③ 자동 초안: 인용 단 초안 생성 (Gemini 도입 시)
                         └ 공통: 환각 방지 · 시행일 표기 · HITL
```

---

## ① 작성 중 — 근거 즉시 조회·인용 (지금 작동)

보고서에 "제조업 인허가" 단락을 쓰다가:

```bash
python scripts/report.py "의료기기 제조업 허가는 누구에게 받고 결격사유는?"
```
결과(실제 출력):
> **[의료기기법 제6조 (시행 2025-08-01)]** 「제3장 제조 등 > 제6조(제조업의 허가 등)」
> ① 의료기기의 제조를 업으로 하려는 자는 **식품의약품안전처장의 제조업허가를 받아야 한다**…
> 출처: https://www.law.go.kr/법령/의료기기법/제6조

→ 이 인용을 **그대로 보고서에 붙여넣기**. law.go.kr을 헤매거나 조항을 잘못 적을 위험 제거.

## ② 작성 후 — 규정 준수 점검 (대조)

이미 쓴 단락의 주장을 지식베이스에 대조 → **근거 있음 / 확인 불가 / 옛 조항** 판정.
- 근거 없는 주장 → `report.py`가 **"확인 불가"**로 표시(지어내지 않음).
- 입찰제안서의 요구사항 충족 대조도 같은 원리(→ M3 컴플라이언스 매칭).

## ③ 자동 초안 — 인용 단 초안 (Gemini 도입 시)

질문 → 지식베이스 검색 → **LLM이 인용을 달아 초안 작성**. 단 인용·확인불가·HITL 규칙은 유지.
(로컬 PoC는 LLM 없이 "근거 원문 제시"까지. 자연어 요약은 Gemini 키 도입 시 부착.)

---

## 버전관리가 "기준의 신뢰"를 만든다

이번에 보강한 부분(`chunk.py`가 시행일로 `is_current`/`status` 계산):

- 모든 인용에 **시행일**이 찍혀, "**지금 유효한 조문**"만 기준이 된다.
- 미래 개정본은 `status="시행예정"`, `is_current=false`로 구분 → **현행으로 오인 인용 방지**.
- 검색을 `--current-only`로 걸면 **현행만** 근거로 삼는다.

**실증 사례(검증 문서 참조):** 의료기기법은 **2026-12-10 시행 개정**으로 제28조가
"품질관리심사기관 지정 등" → "적합성인정"으로 바뀐다. 버전관리가 있으면:
- 현행 보고서엔 **현행본(2025-08-01)** 을 인용하고,
- 두 버전을 함께 적재하면(다음 단계) *"2026-12-10부터 개정 — 향후 대비"* 를 **자동 표시**.

> 현재 구현: 단일 버전 내 현행/시행예정 판정까지 완료. **버전 간 자동 비교 경고**(현행↔개정 예정 diff)는
> 현행+시행예정 두 버전을 함께 적재하는 M2에서 켜진다.

## Before / After

| 항목 | 지금(수작업) | 이 시스템 |

| 근거 찾기 | law.go.kr 검색·스크롤 | 질문 한 줄 |

| 검증 | 전부 수동 | 근거 없으면 "확인 불가" |

## 성숙도 단계 (활용 편의)

| 단계 | 인터페이스 | 상태 |
|------|-----------|------|
| 1 | CLI(`report.py`) — 질문→인용 | ✅ 현재 |
| 2 | + 의미검색(pgvector) + LLM 요약 초안 | ⬜ (Supabase/Gemini 키) |
| 3 | 웹 UI / 보고서 편집기 플러그인(쓰면서 옆에서 인용 추천) | ⬜ 향후 |
| 4 | 입찰 요구사항 자동 대조 매트릭스 | ⬜ M3 |

**규정이 늘수록(M2: 시행령·시행규칙·고시) 기준선이 두꺼워져**, 어떤 사안이든 근거 조항을 즉시 댈 수 있다.
