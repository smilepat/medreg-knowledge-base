작성일: 2026-06-23  |  대상 저장소: smilepat/medreg-knowledge-base  |  분석 범위: 코드·문서·커밋 이력(44개)·실제 데이터

1. 목적 — 무엇을, 왜

한 줄 정의: 의료기기 관련 정부 규정(법령·고시·조달 규정)을 LLM이 잘 쓰는 구조화 지식베이스로 가공해, 보고서·입찰제안서의 규정 준수를 RAG(검색증강생성)로 자동 지원하는 시스템.

핵심 철학: "PDF를 통째로 LLM에 던지기"가 아니라 조항 하나하나를 색인 카드로 잘라 라벨(메타데이터)을 붙여 정리한다. 가공(Phase 1)에 가치의 90%가 있다고 명시한다.

배경: (주)대화기기(Daiwha) AX 컨설팅 트랙과 연계 가능한 실무 도구. 첫 타깃은 보고서용 Q&A 검색, 다음 단계가 입찰 컴플라이언스 매칭.

3대 안전장치 (전 단계 관통)

장치내용
환각 방지검색된 원문이 있을 때만 규정을 단정. 근거 없으면 "확인 불가". 인용엔 법령명+조항+시행일 의무.
버전 관리시행일 기준 "현행" 추적(is_current). 폐지 조항 인용 금지.
HITL (사람 검토)최종본은 사람이 출처 확인. LLM은 초안 작성자이지 결재자가 아님.

2. 개발 과정 — 커밋 44개, 2026-06-15 ~ 06-18 (약 4일)

Claude Code로 한 스크립트씩 만들고 검증 후 다음으로 진행한 마일스톤 방식:

날짜마일스톤내용

6/16M2시행령·시행규칙·GMP고시·국민건강보험법 적재.

6/17배포 준비멀티 PC 셋업(setup_check) + 비개발자 가이드 + app.py(Streamlit 웹앱).
6/18배포 완성클라우드 배포(로그인 보호/Secrets) + 사내망 원클릭(run-app.bat) + Streamlit Community Cloud 대응 + 사용 가이드 팝업.

파이프라인 (4 Phase)

범위 정의  →  원문 → 구조화 (★ 가치의 90%)  →  저장 + 검색 (pgvector)  →  활용

데이터 처리 방식 (실제 확인)

  • 법령 위계(조/항/호/목)를 청킹 경계로 사용 — 청크 1개 = 조항 1개.
  • 각 청크는 context_header(법령 > 장 > 조)와 메타데이터(law_name / article / effective_date / reg_type / is_current / source_url / cross_refs / defined_terms)를 자기완결적으로 포함.
  • 정의어는 definitions 사전으로 분리, 교차 참조는 링크로 연결.
/raw·/processed는 gitignore(용량·기밀)이고, /samples에는 7개 법령의 검증용 첫 3조항 샘플만 들어 있다. 실제 전체(앱 안내 기준 7개 법·584조항)는 Supabase pgvector에 적재되어 있어 로컬 레포만으로는 보이지 않는다.

기술 스택

Python 파이프라인 · Gemini(임베딩/판정) · Supabase pgvector(ai-english-platform 인스턴스 재사용, 별 스키마) · Streamlit UI. 추출 도구(Upstage / LlamaParse / Docling)와 임베딩 모델은 미확정 상태로 기록됨.

3. 사용 방법

A. 웹앱 (권장 · 비개발자용)

pip install -r requirements.txt
streamlit run app.py        # 또는 run-app.bat 더블클릭

사전 준비: .env에 SUPABASE_DB_URL + GEMINI_API_KEY (python scripts/setup_check.py로 점검). 브라우저(localhost:8501)에서 3개 탭:

탭기능사용
📖 규정 근거찾기"이거 무슨 법 몇 조야?"질문 입력 → 관련 조항을 출처·시행일·유사도와 함께. 유사도 0.50 미만이면 "확인 불가".
✅ 문서 점검"내 문서가 규정에 맞나?"문장을 한 줄에 하나씩 → ✅근거있음 / ⚠️충돌 / ⚠️근거없음 (LLM 판정 옵션).
📋 입찰 매트릭스"입찰 요건을 충족하나?"요구사항 + (선택)회사 프로필 → 유형·규정근거·충족 표.

사이드바에서 "현행 규정만" 필터 토글. APP_PASSWORD 설정 시 로그인 게이트.

B. CLI 파이프라인 (개발 / 데이터 확장용)

python scripts/fetch_law.py "의료기기법"                       # ① Open API로 구조화 XML 수집 (LAW_OC 키 필요)
python scripts/structure.py "raw/의료기기법.xml"               # ② 위계 트리 JSON+MD
python scripts/chunk.py "processed/..._structured.json"        # ③ 조 단위 청킹+메타데이터
python scripts/embed.py ...                                    # ④ Gemini 임베딩 → pgvector 적재
python scripts/search.py "의료기기 등급분류 기준" --top 3        # ⑤ 검색
python scripts/report.py "제조업 허가는 누구에게 받나?"           # ⑥ 인용 답변 초안

주요 스크립트 12종

스크립트역할

한 줄 요약

4일간 한 스크립트씩 쌓아 만든 한국 의료기기 규정 RAG 비서 — 법령 위계를 보존한 청킹과 3대 안전장치(환각방지·버전관리·HITL)로 "지어내지 않고 출처와 함께 답하는" 것을 핵심 가치로 삼으며, CLI 파이프라인 위에 Streamlit 3탭 웹앱과 사내망/클라우드 배포까지 갖춘 PoC 단계 도구.
