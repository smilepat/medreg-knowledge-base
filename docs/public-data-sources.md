# 의료기기 사업 관련 법령·규제 공공 출처 (조사: 2026-06-16)

> 의료기기 사업의 규제 영역별로 법령·규제 텍스트를 받을 수 있는 공공 출처.
> **우리 파이프라인 활용도(자동 수집 가능 여부)** 기준 우선순위로 정리.
> ⚠️ 법령 텍스트는 공공저작물(자유 이용)이나, 가이드라인 PDF·DB는 약관·저작권 확인 필요.
> 자동 스크래핑은 약관 위반 위험 → **공식 Open API / 공개 다운로드** 우선.

## Tier 1 — 법령 "본문" (우리 파이프라인 직접 입력, 최우선)

| 사이트 | 제공 | 수집 방식 |
|--------|------|-----------|
| **국가법령정보센터** law.go.kr / open.law.go.kr | 법·시행령·시행규칙 + **행정규칙(고시·훈령·예규)** + **의료기기 기준규격** 전문 | ✅ **Open API**(`fetch_law.py`가 사용). `target=law`/`target=admrul` |
| **법제처 입법예고** moleg.go.kr | 의료기기법/시행령/시행규칙 **개정(안) 예고** | ⭐ **시행예정 추적**(버전관리 데이터원) |
| **공공데이터포털** data.go.kr | 법제처 국가법령정보 공유서비스 + 의료기기 허가/UDI Open API | ✅ API |

> 핵심: 의료기기 규제 "텍스트" 대부분(법·령·규칙·고시·기준규격)은 **law.go.kr Open API 하나로 자동 수집**.
> `fetch_law.py`가 `target=law`(법령) + `target=admrul`(고시) 둘 다 지원함(2026-06-16 구현).

## medreg 적용 권고

1. **자동 수집(우선)**: Tier 1 전부 + 식약처 고시는 **law.go.kr Open API 한 곳**에서.
   `fetch_law.py --target admrul`로 고시 자동 수집 가능(구현 완료).
2. **시행예정 추적**: 법제처 입법예고(moleg) → "현행+시행예정" 경고의 데이터원.
3. **수동 다운로드(보조)**: 식약처 민원인 안내서·가이드라인(PDF)은 해석 기준 → `extract.py`(표 보존) 경로.
4. **도메인 확장 순서**: 인허가(법·고시) → 품목분류/UDI → 보험수가(HIRA) → 조달(나라장터).
   제조사(예: 대화기기)는 Tier 1~3가 핵심, 입찰 비중 크면 Tier 4.
5. **2차 소스**: legalize-kr(IP·키 無, [verification-legalize-kr.md](verification-legalize-kr.md) 참조).
