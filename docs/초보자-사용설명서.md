# 왕초보 사용 설명서 — 천천히 따라 하기

> 컴퓨터를 잘 몰라도 괜찮습니다. **그대로 따라 하면** 됩니다.
> 어려운 말은 풀어서 설명했고, 복사해서 붙여넣을 부분은 회색 상자에 넣어 뒀습니다.

---

## 0. 이 프로그램은 뭔가요?

의료기기와 관련된 **법·규정 7개(조항 584개)를 다 외우고 있는 비서**라고 생각하세요.
- 질문하면 → "○○법 몇 조에 이렇게 적혀 있어요"라고 **출처까지** 알려줍니다.
- 내가 쓴 문서를 주면 → "이 문장은 규정에 맞아요 / 어긋나요"를 표로 알려줍니다.
- 입찰 공고문을 주면 → "이 요건은 무슨 규정 근거이고, 우리가 충족하나요"를 정리해 줍니다.

> ⚠️ 단, 이 비서의 답은 **초안**입니다. 중요한 결정은 꼭 사람이 원문을 확인하세요.

## ★ 가장 쉬운 방법 — 웹 화면(브라우저)으로 쓰기

명령어를 매번 치기 어렵다면, **웹 화면 한 번 띄워 놓고 클릭·입력**만 하면 됩니다.

1. 아래 1번처럼 명령창(PowerShell)을 엽니다.
2. 딱 한 줄만 입력하고 **Enter**:
   ```
   python -m streamlit run app.py
   ```
3. 잠시 뒤 **인터넷 브라우저가 자동으로 열립니다**(주소: http://localhost:8501).
4. 화면 위 **세 개의 탭**을 클릭해서 사용:
   - **📖 규정 근거찾기** — 질문 입력 → [근거 찾기] 클릭
   - **✅ 문서 점검** — 문장 붙여넣기 → [점검하기] 클릭
   - **📋 입찰 매트릭스** — 공고문(+회사정보) 붙여넣기 → [매트릭스 생성] 클릭
5. 다 쓰면 그 명령창을 닫으면 종료됩니다.

> 이 방법은 명령창을 **처음 한 번만** 쓰고, 그 다음부턴 전부 브라우저에서 클릭·입력입니다.
> 아래 2~4번은 웹 화면 없이 **명령어로 직접** 쓰는 방법입니다(원하는 분만).

## 1. 한 번만 하면 되는 준비

### (1) '명령어 입력 창' 열기

이 프로그램은 아이콘을 더블클릭하는 방식이 아니라, **글자로 명령을 입력**해서 씁니다.
그 입력 창을 여는 방법:

1. 화면 아래 **파일 탐색기**(노란 폴더 모양 아이콘)를 엽니다.
2. 다음 폴더로 들어갑니다:
   `내 PC → C: → 사용자(Users) → user → Documents → GitHub → medreg-knowledge-base`
3. 폴더 안의 빈 곳을 보면 위쪽에 **주소 칸**(폴더 경로가 적힌 긴 칸)이 있습니다.
   그 주소 칸을 마우스로 한 번 클릭해서 글자를 지우고, 아래를 입력한 뒤 **Enter**:
   ```
   powershell
   ```
4. 그러면 **파란색(또는 검은색) 글자 창**이 뜹니다. 이게 명령어 입력 창입니다. (이름: PowerShell)

> 이 창을 닫지 말고 아래 작업을 이어서 하세요. 닫았으면 1번부터 다시 열면 됩니다.

### (2) 한글이 깨지지 않게 (매번 창을 새로 열 때마다 1회)

입력 창에 아래를 복사해 붙여넣고 **Enter**:
```
$env:PYTHONIOENCODING="utf-8"
```
(붙여넣기는 입력 창 안에서 **마우스 오른쪽 클릭**하면 됩니다.)

### (3) 준비됐는지 확인

아래를 붙여넣고 **Enter**:
```
python scripts/setup_check.py
```
- 화면에 **✅** 표시들이 나오면 준비 끝입니다.
- **❌**가 보이면 담당자(또는 Claude)에게 그 화면을 그대로 보여주세요.

## 2. 사용법 ① — 규정 근거 물어보기

"이거 무슨 법 몇 조에 있지?"가 궁금할 때.

입력 창에 아래를 붙여넣되, **큰따옴표 안의 질문만 바꾸면** 됩니다. 그리고 **Enter**:
```
python scripts/report.py "의료기기 제조업 허가는 누구에게 받나?"
```

**결과 읽는 법** — 잠시 기다리면 이런 식으로 나옵니다:
```
■ 관련 규정 (근거):
1. [의료기기법 제6조 (시행 2025-08-01)] ...
   ① 의료기기의 제조를 업으로 하려는 자는 식품의약품안전처장의 제조업허가를 받아야 한다 ...
   출처: https://www.law.go.kr/법령/의료기기법/제6조
```
- 대괄호 `[ ]` 안 = **인용할 법령·조항·시행일**. 보고서에 그대로 적으면 됩니다.
- `출처:` 줄 = 원문 주소. 확인하고 싶으면 그 주소를 인터넷 창에 붙여넣으세요.
- 만약 **"⚠️ 확인 불가"**라고 나오면 → 그 내용은 이 비서가 가진 규정에 없다는 뜻입니다(지어내지 않음).

질문을 바꿔 가며 여러 번 물어봐도 됩니다. 예:
```
python scripts/report.py "의료기기 등급은 어떻게 나누나?"
```

## 3. 사용법 ② — 내가 쓴 문서 점검하기

내 보고서가 규정과 어긋나지 않는지 확인할 때.

### (1) 점검할 문서를 만들기
1. **메모장**을 엽니다 (화면 아래 돋보기 클릭 → `메모장` 입력 → 실행).
2. 점검하고 싶은 문장을 **한 줄에 하나씩** 적습니다. 예:
   ```
   의료기기 제조업 허가는 식품의약품안전처장에게 받는다.
   우리 제품은 의료기기법 제15조에 따라 제조허가를 받았다.
   ```
3. **파일 → 저장**. 저장 위치는 `medreg-knowledge-base` 폴더 안,
   파일 이름은 `내문서.txt` 로 합니다.

### (2) 점검 실행
입력 창에 붙여넣고 **Enter**:
```
python scripts/check.py 내문서.txt --out 점검결과.md
```

### (3) 결과 보기
- 잠시 후 폴더 안에 **`점검결과.md`** 파일이 생깁니다. 더블클릭해서 엽니다.
- 표로 정리돼 있습니다:
  - **✅ 근거있음** = 규정에 맞음
  - **⚠️ 충돌(모순)** = 규정과 어긋남 (예: 잘못된 조항 인용)
  - **⚠️ 근거없음** = 뒷받침할 규정을 못 찾음 → 확인 필요
- ⚠️ 표시가 있는 줄은 담당자가 한 번 더 확인하세요.

## 4. 사용법 ③ — 입찰 요구사항 대조하기

입찰 공고문의 요구조건을 규정·회사 역량과 맞춰볼 때.

### (1) 두 개의 파일 준비
1. **공고문 파일**: 메모장에 입찰 요구사항을 한 줄씩 적어 `공고문.txt`로 저장.
2. **회사 정보 파일**(선택): 예시 파일 `examples\company-profile.sample.md`를 열어 보고,
   우리 회사 내용으로 바꿔 `examples\company-profile.md`로 저장하면 충족 여부까지 봐줍니다.

### (2) 실행
회사 정보 없이 규정 근거만:
```
python scripts/bid_matrix.py 공고문.txt --out 입찰표.md
```
회사 정보까지 대조(충족 여부 표시):
```
python scripts/bid_matrix.py 공고문.txt --company examples/company-profile.md --out 입찰표.md
```

### (3) 결과 보기
- 폴더에 생긴 **`입찰표.md`**를 더블클릭.
- 요구사항별로 **유형 · 규정 근거 · 충족(✅/❌/⚠️)** 이 표로 정리됩니다.

## 5. 자주 나오는 문제와 해결

| 이런 일이 | 이렇게 하세요 |
|-----------|----------------|
| 글자가 ▒▒▒처럼 깨져 보임 | 1-(2)의 `$env:PYTHONIOENCODING="utf-8"` 를 먼저 입력 |
| "python을(를) 찾을 수 없습니다" | 파이썬이 설치 안 됨 → 담당자에게 설치 요청 |
| ❌ 표시가 나옴 | 그 화면을 그대로 캡처해 담당자/Claude에게 전달 |
| 결과가 영 이상함 | 답은 '초안'입니다. **출처 주소의 원문을 직접 확인**하세요 |
| 파일을 못 찾는다고 나옴 | 파일을 `medreg-knowledge-base` 폴더 안에 저장했는지 확인 |
