# 추출 도구 비교 평가 (M1-1)

> 한국 정부 문서 함정: ① HWP(한글) 多 ② 표 多 ③ 스캔본(이미지 PDF) 혼재.
> 일반 텍스트 추출은 표를 망가뜨린다 → 표 보존이 핵심 평가 기준.

## 평가 기준
- **표 보존**: 표가 Markdown 표/구조로 살아남는가 (★ 최우선)
- **HWP 지원**: 한글 파일 직접/변환 처리
- **OCR**: 스캔본(이미지 PDF) 처리
- **한국어 품질**: 한국어 레이아웃·법령 구조 인식
- **비용**: API 과금 vs 로컬 무료
- **로컬 실행**: 오프라인/온프레미스 가능 여부

## 후보

| 도구 | 표 보존 | HWP | OCR | 비용 | 로컬 | 비고 |
|------|:---:|:---:|:---:|------|:---:|------|
| **Upstage Document Parse** | ? | ? | ✅ | API 과금 | ❌ | 한국 업체, 한국어 강점 기대 |
| **LlamaParse** | ? | ? | ✅ | API 과금(무료 한도) | ❌ | LLM 기반 파싱 |
| **Docling** (IBM) | ? | ? | ✅ | 무료 | ✅ | 오픈소스, 로컬 실행 |

> `?` = 샘플 1건으로 직접 검증 필요. M1-1에서 동일 샘플 PDF로 3종 돌려 표 보존 비교 후 채움.

## HWP 처리 메모
- HWP는 별도 변환 경로 필요 (예: hwp5 → html/text, 또는 한컴오피스 PDF 변환 후 처리).
- 변환 단계에서 표가 깨지기 쉬움 → 변환 직후 표 보존 검증 필수.

## 결정 (채움 예정)

- 근거: _샘플 검증 후 작성_
