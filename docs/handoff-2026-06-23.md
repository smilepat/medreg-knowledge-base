## 1. 목표

### 이번 세션 목표
- 키(Supabase/Gemini) 없이도 **로컬에서 즉시 쓸 수 있는 사용 경로**를 확보한다.
- 다른 PC에서 작업을 이어갈 수 있도록 환경/실행법을 문서화한다.

### 배경(왜 했나)
`app.py`(원본 Streamlit 웹앱)는 [retriever.py](../scripts/retriever.py)가 **전적으로
Supabase pgvector + Gemini 임베딩에 의존**한다. 그런데 로컬 `.env`는 `.env.example`을
복사만 한 **placeholder 상태**(호스트명이 가짜 `aws-...pooler.supabase.com`)라,
앱이 시작 시 법령 목록을 불러오려 DB에 붙다가 `psycopg2.OperationalError`로 죽었다.
→ 키 없이 동작하는 로컬 경로가 필요했다.

## 4. 다른 PC에서 이어서 작업하는 법

### 4.1 공통 셋업
```bash
git clone https://github.com/smilepat/medreg-knowledge-base.git
cd medreg-knowledge-base
python -m pip install -r requirements.txt
```

### 4.2 로컬 경로(키 없이 바로 사용) — 권장 시작점
```bash
# 웹 UI (브라우저 자동 열림, http://localhost:8501)
python -m streamlit run app_local.py

# 또는 CLI 한 줄 검색
python ask.py "의료기기 제조업 허가"
```
- 추가 설정 불필요. `samples/`만 있으면 동작.

### 4.3 풀 기능 경로(`app.py`) 켜려면
1. `.env` 생성: `.env.example`를 복사해 실제 값 채우기.
   - `SUPABASE_DB_URL=postgresql://USER:PASS@<진짜호스트>.pooler.supabase.com:5432/postgres`
   - `GEMINI_API_KEY=<aistudio.google.com/apikey 발급>`
   - (원문 재수집 시) `LAW_OC=smilepat` + 호출 PC 공인 IP를 open.law.go.kr에 등록
2. 점검: `python scripts/setup_check.py`
3. DB 적재(최초 1회): `python scripts/embed.py` → Supabase `medreg.chunks`에 임베딩 적재
4. 실행: `python -m streamlit run app.py`

> ⚠️ `.env`는 git에서 제외된다(`.gitignore`). 키는 PC마다 직접 넣어야 한다.
